# Project Profile

## Purpose

Create a reusable `devgod` platform that lets Codex treat substantive requests like customer or CEO asks, decompose them into safe task packets, coordinate specialist agents, and turn them into secure, clean, tested implementations across multiple projects.

## Users

- primary: repo owner guiding product direction
- secondary: future Codex threads working on the same project
- tertiary: teams reusing one shared orchestration backend

## Non-negotiables

- secure by default
- clear architecture
- strong UX for user-facing products
- token-efficient agent coordination
- durable local memory
- shared backend for operational state and retrieval

## Current Status

- shared-core runtime foundation added
- specialist agents defined
- planner role added
- project-scoped skills, rules, templates, and work artifacts defined
