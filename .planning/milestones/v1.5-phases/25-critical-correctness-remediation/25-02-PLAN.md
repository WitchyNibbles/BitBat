<objective>
Fix two production code quality issues: regression_metrics() mixing computation with file I/O side effects (CORR-03), and assert isinstance in production code that gets stripped under python -O (CORR-04).

Purpose: regression_metrics() currently writes metrics/regression_metrics.json and metrics/prediction_scatter.png as hidden side effects, violating function purity. The assert isinstance guards in train.py would silently pass wrong types if Python runs with -O flag. Both are correctness risks.

Output: Refactored evaluate.py, fixed train.py, 2 new test files
</objective>
