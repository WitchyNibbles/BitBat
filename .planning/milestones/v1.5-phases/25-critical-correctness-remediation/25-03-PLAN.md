Create `tests/features/test_leakage.py` with `pytestmark = pytest.mark.behavioral`.

This test file implements the PR-AUC guardrail referenced in CLAUDE.md (`tests/features/test_leakage.py -- no future data in features`). The test detects train/test information leakage by checking that a model trained on random labels cannot achieve suspiciously high PR-AUC, which would indicate the features contain future information.

**Test structure:**

1. **test_no_feature_leakage_pr_auc_guardrail**: The primary guardrail test.
   - Generate a small synthetic price dataset (200+ bars of realistic BTC-like prices using numpy random walk with seed=42).
   - Create a DataFrame with columns: open, high, low, close, volume (all with a DatetimeIndex at 1h frequency).
   - Call `_generate_price_features()` from `bitbat.dataset.build` to produce the feature matrix.
   - Drop NaN rows from features.
   - Create random binary labels (0/1) using the same seed -- these are intentionally random and should NOT be predictable from features.
   - Split into train/test using a simple time-based split (first 70% train, last 30% test) -- no walk-forward needed for this guardrail.
   - Train a quick XGBoost model on train features + random labels.
   - Predict on test features.
   - Compute PR-AUC (precision-recall area under curve) using `sklearn.metrics.average_precision_score`.
   - Assert PR-AUC < 0.7 (a model predicting random labels should get ~0.5 PR-AUC; anything above 0.7 indicates leakage).

2. **test_features_no_future_timestamps**: Verify that for each row in the feature matrix, all feature values are computable from data at or before that row's timestamp. This checks that:
   - Lagged returns only use past data (lag >= 1).
   - Rolling windows only look backward.
   - OBV cumsum uses data up to current bar only.
   Implementation: Generate features for a known dataset, then verify that re-computing features on a truncated dataset (first N rows) produces the same values for row N as the full dataset.

3. **test_obv_no_lookahead**: Specifically test the OBV feature:
   - Create a price series of 100 bars.
   - Compute OBV for all 100 bars.
   - Compute OBV for first 50 bars only.
   - Assert the OBV values for bars 0-49 are identical in both computations.
   This is a targeted check for the cumsum leakage concern (LEAK-01 related, but this tests the basic no-lookahead property).

Use `pytest.importorskip("sklearn")` for the PR-AUC test since sklearn may not be in all environments.

**Create a shared default-value helper:**

At the top of each API route file (or in a shared location like `src/bitbat/api/defaults.py`), create a helper that reads freq/horizon from config:

```python
# src/bitbat/api/defaults.py
from bitbat.config.loader import load_config

def _default_freq() -> str:
    config = load_config()
    return str(config.get("freq", "1h"))

def _default_horizon() -> str:
    config = load_config()
    return str(config.get("horizon", "4h"))
```

Note: Using `load_config()` (not `get_runtime_config()`) to avoid depending on cached mutable state. The fallback values "1h"/"4h" in the `.get()` are last-resort defaults if the config file is missing both the key AND the default.yaml.

**Update all hardcoded defaults in API routes:**

In `src/bitbat/api/routes/predictions.py`, replace all instances of `Query("1h"...)` and `Query("4h"...)` with calls to the config defaults. FastAPI's `Query` default parameter needs a concrete value at import time, so use a pattern like:

```python
from bitbat.api.defaults import _default_freq, _default_horizon

# At module level (computed once at import)
_FREQ = _default_freq()
_HORIZON = _default_horizon()

@router.get("/latest", response_model=PredictionResponse)
async def latest_prediction(
    freq: str = Query(_FREQ, description="Bar frequency"),
    horizon: str = Query(_HORIZON, description="Prediction horizon"),
) -> PredictionResponse:
```
