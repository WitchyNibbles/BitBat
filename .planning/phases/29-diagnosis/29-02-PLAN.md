Bug inventory (all confirmed HIGH confidence by code inspection + live DB):

Bug 1 (PRIMARY — Model stage):
  File: src/bitbat/model/train.py, line 53
  Bug: objective = "reg:squarederror" — regression objective trains on r_forward float, not class labels
  Evidence: xgb.Booster.save_config() on both model artifacts returns "reg:squarederror"
  Impact: Model predicts continuous return magnitude; output is biased negative (mean = -0.00246)
  Reproducible trace: poetry run pytest tests/diagnosis/test_pipeline_stage_trace.py::test_model_objective_is_regression -v

Bug 2 (SECONDARY — Serving/inference stage):
  File: src/bitbat/model/infer.py, line 86
  Bug: predicted_direction = "up" if predicted_return > 0 else "down" — no tau threshold, no "flat" class
  Evidence: 203/268 predictions labeled "down" (76%), 0 labeled "flat"
  Impact: Heavily biased toward "down" because regression output is negative-biased
  Reproducible trace: poetry run pytest tests/diagnosis/test_pipeline_stage_trace.py::test_serving_direction_bias -v

Bug 3 (TERTIARY — Validation/realization stage):
  File: src/bitbat/autonomous/validator.py, line 46
  Bug: self.tau = 0.0 hardcoded, overrides constructor param; price lookup returns actual_return = 0.0 for 67% of rows
  Evidence: 179/266 realized predictions have actual_return = 0.0 exactly; classify_direction with tau=0.0 yields zero "flat" from non-zero moves
  Impact: actual_direction = "flat" for majority of rows (from price data gaps); "down" predictions never match "flat" actuals
  Reproducible trace: poetry run pytest tests/diagnosis/test_pipeline_stage_trace.py::test_validation_zero_return_corruption -v

Compounding effect:
  Bug 1 produces negative-biased regression outputs.
  Bug 2 maps those to 76% "down" predicted_direction.
  Bug 3 produces 67% "flat" actual_direction (from zero actual_return via price lookup gaps).
  Result: "down" vs "flat" = wrong. Overall: 38/266 = 14.3% (vs 33% random baseline).

Fix phase: Phase 30 (all three bugs fixed in code + model retrain).
