## BitBat Local Happy Path

This guide walks through a full \"happy path\" on a single machine — from first run to confirming that models and history persist across restarts.

---

## 1. Install and launch the stack

1. Install dependencies (from the project root):
   - `poetry install`
2. Start the autonomous stack:
   - `make autonomous-up`
3. Open the React dashboard at `http://localhost:3000`.

---

## 2. Confirm the runtime bootstrap

1. `make autonomous-up` runs `scripts/ensure_autonomous_stack_ready.py` before the services come up.
2. Confirm a runtime artifact exists under `models/{freq}_{horizon}/`.
3. Confirm `data/autonomous.db` contains at least one active model row after bootstrap.

---

## 3. Let continuous monitoring run

1. The compose stack starts ingestion and monitoring automatically.
2. Confirm the services stay healthy and leave them running for a few monitoring cycles so predictions and validations accumulate.

---

## 4. Inspect performance and history

After some time has passed:

1. Open the React dashboard and inspect the runtime and Oracle views.
2. Confirm that predictions, recent outcomes, and runtime status become non-empty over time.

All of this data is read from `data/autonomous.db`, using realized predictions generated by the monitoring loop.

---

## 5. Restart and confirm persistence

1. Stop the stack:
   - `make autonomous-down`
2. Verify that the `data/` and `models/` directories still exist:
   - `data/autonomous.db`
   - `data/raw/**`
   - `data/features/**`
   - `models/{freq}_{horizon}/`
3. Restart the stack:
   - `make autonomous-up`
4. Revisit the dashboard:
   - The same predictions, runtime history, and active model state should still be available.

If the dashboard appears empty after restart, consult `docs/persistence-overview.md` and ensure you are running from the same project root with the original `data/` and `models/` folders.
