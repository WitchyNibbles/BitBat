## BitBat Local Happy Path

This guide walks through a full \"happy path\" on a single machine — from first run to confirming that models and history persist across restarts.

---

## 1. Install and launch the app

1. Install dependencies (from the project root):
   - `poetry install`
2. Start the Streamlit UI:
   - `poetry run streamlit run streamlit/app.py`

---

## 2. Train a model with Quick Start

1. Open the **Quick Start** page in the sidebar.
2. Choose a preset (for example **Balanced**) and click **Train Model**.
3. Wait for training to complete; you should see a success banner showing:
   - model version,
   - number of training samples,
   - training duration.
4. This creates:
   - a model file under `models/{freq}_{horizon}/xgb.json`,
   - a corresponding `model_versions` row in `data/autonomous.db`.

---

## 3. Start continuous monitoring

1. Still on **Quick Start**, scroll to the **Monitoring Status** section.
2. Click **Start Monitoring**.
3. Confirm that:
   - the **Monitoring Status** metric shows **Running**,
   - the **Validation Interval** matches your configuration.
4. Leave the app running for at least a few monitoring cycles so predictions and validations accumulate.

---

## 4. Inspect performance and history

After some time has passed:

1. Open the **Performance** page:
   - Check **Overall Statistics** for total/completed/correct predictions.
   - Review **Accuracy Over Time** and the **Retraining History** + **Retraining Impact** expanders.
2. Open the **History** page:
   - Confirm the calendar heatmap and weekly trend show non-empty data.

All of this data is read from `data/autonomous.db`, using realized predictions generated by the monitoring loop.

---

## 5. Restart and confirm persistence

1. Stop the Streamlit process (Ctrl+C).
2. Verify that the `data/` and `models/` directories still exist:
   - `data/autonomous.db`
   - `data/raw/**`
   - `data/features/**`
   - `models/{freq}_{horizon}/xgb.json`
3. Restart the app:
   - `poetry run streamlit run streamlit/app.py`
4. Revisit the **Performance** and **History** pages:
   - The same predictions, accuracy metrics, and retraining history should be visible.
5. On **Quick Start**, you can:
   - See the existing active model information,
   - Start monitoring again with **Start Monitoring** if desired.

If the pages appear empty after restart, consult `docs/persistence-overview.md` and ensure you are running from the same project root with the original `data/` and `models/` folders.
